## Phase 1: Immediate Fixes (broken/half-working)

### 1. Fix webhook channel names

- **Files:** `src/app/api/webhooks/neynar/route.ts`

### 2. Add missing notification triggers
- **Problem:** Notifications only fire from 4 places. No triggers for XMTP DMs, new member joins, reactions, respect.
- **Files:** `src/app/api/webhooks/neynar/route.ts`, `src/app/api/auth/verify/route.ts`, `src/app/api/chat/react/route.ts`
- **Effort:** 1-2 hours

### 3. Cache thread replies in webhook
- **Problem:** `if (cast.parent_hash) return` skips all replies. Threads always hit Neynar live.
- **Files:** `src/app/api/webhooks/neynar/route.ts`, `src/app/api/chat/thread/[hash]/route.ts`
- **Effort:** 1-2 hours

### 4. Replace notification polling with Supabase Realtime
- **Problem:** Polls `/api/notifications` every 30s. 40 users = 80 queries/minute.
- **Files:** `src/components/navigation/NotificationBell.tsx`, `src/lib/db/supabase.ts`
- **Effort:** 2-3 hours
- **Dependency:** Browser-safe Supabase client (Phase 4, item 15)

## Phase 4: Architecture Improvements

### 14. Add React Error Boundaries
- **Problem:** No error boundaries anywhere. Any component throw crashes the entire page.
- **Files:** `src/app/(auth)/layout.tsx`, wrapper around XMTP/music providers
- **Effort:** 2-3 hours

### 15. Create browser-safe Supabase client
- **Problem:** Only `supabaseAdmin` exists (service role, server-only). Blocker for Realtime features.
- **Files:** `src/lib/db/supabase.ts`
- **Effort:** 30 minutes
- **Note:** Requires `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY` env vars

### 16. Refactor ChatRoom.tsx
- **Problem:** 644 lines, 20+ state variables. God component managing everything.
- **Scope:** Extract `ChatHeader`, `ChatLayout`, `useXMTPNavigation`, `useMusicPlayback` hooks
- **Effort:** 1-2 days

### 17. Persistent rate limiter
- **Problem:** In-memory Map resets on every serverless cold start.
- **Scope:** Switch to Vercel KV or Supabase-backed counters
- **Effort:** Half day

### 18. Enable TypeScript strict mode
- **Problem:** `any` types, no strict null checks. SDK type changes go unnoticed.
- **Scope:** Enable `strict: true`, fix errors incrementally (lib → hooks → components)
- **Effort:** 1-2 days
