**The thesis that emerged:** Flat RAG (vector search over 574 docs) is a retrieval tool. A knowledge graph with typed relationships is a thinking tool. **Doc 545 — ZABAL KG Ontology v1** — built the node/edge schema for $ZABAL's knowledge graph: Community nodes (members, roles, Respect scores) → Token nodes (ZABAL, contracts, staking) → Product nodes (ZAOstock, ZAO Music, WaveWarZ) → Research nodes (all 574 docs). This isn't a filing system. It's a reasoning layer.

**$ZABAL chain decision research** is the week's biggest strategic output. Two docs:

*Doc 572 — Chain Decision:* Avalanche L1 gives sovereignty (no sequencer fees, custom gas token, own governance). Base gives distribution (Coinbase ecosystem, bridging UX, name recognition). AVAX L2 is the middle path (AVAX security + some sovereignty). The decision matrix: if ZAO's primary user is a music creator, Base wins on UX. If ZAO's primary value is community sovereignty and on-chain governance, Avalanche L1 wins. If ZAO needs both, AVAX L2 is the answer.

*Doc 573 — AVAX Ecosystem:* Arena Social Layer (gated communities + $ZABAL token integration), Retro9000 retroactive grants (active program, ZAO-eligible), music royalty rails with AVAX settlement. These are the ecosystem hooks that don't exist on Base.
