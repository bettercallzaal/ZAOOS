## What Shipped

### ZAO Estate Control Plane (PR #848) — Friday

The week's largest ship. Automated truth-keeping for the whole stack.

**What it does:** One `run-checks` CLI compares what `CLAUDE.md`/`AGENTS.md` say the repo contains against what's actually there. Four checks: drift (component counts vs. live), zombie (decommissioned code still present), quality (untested API domains, npm audit, typecheck), estate (manual-scan freshness). Outputs `estate-report.json`, fans out to PR guardrail + dashboard + Telegram digest.

**Adoption safety:** Ratchet design — existing debt is allowed, new debt blocks PRs. npm audit baseline allowlist (no regressions from pre-existing vulns). Change-only digests (silent when nothing drifted).

**CI:** `.github/workflows/estate-health.yml` — sticky comment on every PR, weekly artifact sweep.

**Verification:** 8 tests passing against a programmatic fixture repo. Correctly caught all 6 real count drifts and the phantom `contracts/` path on first live run.

This is repo-agnostic via `config.json` — portable to Fishbowlz, ZAOstock, and any future graduate.
