## What Shipped

### Scrape Subsystem: 14-Iteration Autonomous Build (PRs #883–#901 + #903)

The lab now has a complete no-login data-collection layer. What started as one X Article reader (doc 873, PR #882) became a full subsystem across 14 autonomous loop iterations:

| Scraper | What it reads | Key method |
|---------|--------------|------------|
| `x-fetch.ts` | Full X Article bodies + tweets | FxTwitter API (no account) |
| WaveWarZ battle history | Full paginated battle history | RSC flight-tree parsing |
| BCZ Farcaster history | All Zaal Farcaster posts | Neynar cast search |
| BCZ site | bettercallzaal.com content | Jina Reader + metadata |
| X user timeline | Any user's timeline | FxTwitter user endpoint |
| Farcaster resolver | @handle → FID | Neynar lookup |
| BCZ profile aggregator | Site + timeline + Farcaster composed | Promise.allSettled |

All wired into a single `/api/scrape` route via a unified dispatcher. Retry-with-backoff on transient failures. Cache persistence to `scrape_cache` table. Zero new external dependencies.

**The key technical findings (captured in doc 880):**
- Reading X is free. Writing needs Premium+. The paywall only guards one direction.
- WaveWarZ data lives in the RSC flight-tree, not the rendered DOM. You have to read the raw JSON.
- Silent 0s in governance data accumulate when you don't fail loudly. Now we do.
- Stacked PRs that all touch the same file compound into merge conflicts — learned this the hard way by iteration 6.

### Silent-Failure Fixes — Governance Correctness

**Respect-sync silent-zero (PR #899):** Failed RPC reads were writing 0 RESPECT balance to Supabase, silently overwriting valid values. No error, no log. Just slowly wrong governance numbers. Fixed: skip the write on failed read.

**WaveWarZ spotlight tier single-source (PR #902):** `getNewSpotlightTier()` maintained a parallel `tierOrder` array alongside the canonical `SPOTLIGHT_TIERS` constant. Match today; diverge on any rename. Fixed: derive the tier index from `SPOTLIGHT_TIERS` directly. 7 tests now cover all paths (thresholds, skip-tier jumps, no-demotion).

The pattern: silent wrongness is the hardest class of problem. It requires looking, not waiting.
