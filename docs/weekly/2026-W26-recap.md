## What Shipped

### ZOL: Concept to Running Agent in 24 Hours

The week started with ZOL as a registered FID and a research question. By Wednesday evening it had three deep docs and working code.

**Doc 891 — Farcaster Agentic Bootcamp synthesis + ZOL build plan (DEEP)**
The synthesis of the Agentic Bootcamp content that produced the concrete build plan. Established ZOL's FID (3338501 / @zolbot) and the four-layer architecture: cast drafting → Bonfire memory recall → x402 payment → Neynar publish.

**Doc 892 — Being an agent on Farcaster 2026 (DEEP)**
The live 2026 landscape for running a Farcaster agent. Key findings that shaped every implementation decision:
- Neynar user score is the real reach gate. Protect it. Agents that cast junk get their score tanked and their reach disappears.
- Mini Apps/Snaps are the native agent surface — not just raw casts.
- Clanker, Bracky, and Aethernet studied as the three agents that have actually won distribution. Common thread: they're useful on first contact, not promotional on first contact.
- EIP-8004 (agent standards): interesting spec, zero Farcaster adoption in 2026, skip for v1.

**Doc 893 — ZOL music-native Farcaster agent + persona (DEEP)**
Defines what ZOL IS, not just what it does. The open lane: no music agent has won Farcaster. Discovery on the protocol is still human-curation-driven. Sound.xyz's mint-collapse is the cautionary tale — when rewards become the product instead of the music, the music disappears. ZOL's one economic action: curate-to-reward. Silence heuristic: only cast when there's something worth sharing. Silence builds weight; chatter destroys it. Persona: ZOL as a ZOE variant, overriding only the persona block. No new architecture.

**PR #957 — feat(zol): Bonfire memory wired into cast drafting**
The first implementation PR. Before drafting any cast, ZOL calls `recall()` against the ZABAL Bonfire graph and injects returned episodes into the OpenRouter system prompt. ZOL casts from ZAO's actual institutional knowledge, not the base model's priors. Degrades gracefully: a graph hiccup never blocks a cast. Three tests, type-clean.

**PR #961 — feat(zol): mention-reply connector (event-stream → approval-gated caster)**
Second implementation PR. Wires Neynar's event stream to ZOL's reply loop, with approval gating before anything goes live. ZOL sees mentions, drafts replies, holds for approval. The agent loop is live; the human stays in the loop.

**PR #958 — fix(farcaster): x402 write price 0.001 → 0.01 USDC**
Neynar raised the x402 write price. Caught and fixed the same day ZOL was shipping. Small fix; would have blocked every cast otherwise.

What ZOL needs to go fully live: Base USDC top-up for x402 writes, profile details (bio, avatar, link), and the first cast. Not an announcement. One song worth sharing.

### Social Voice Codified (Docs 897 + 898)

After a year of building in public, the posting voice is finally in writing.

**Doc 897 — ZAO Social Posting Playbook**
Hook-first structure. Sell the outcome, not the event title. On X in 2026: link in the first reply (body links cut reach ~50%). Farcaster: short, channel-native, no hashtags, tag guests by handle. Optimize for replies and bookmarks, not likes.

**Doc 898 — Zaal Brand Voice for Posts**
ZM opener + hook + outcome + rally close. "Free live recorded" for event posts. Phrase library. Never-say list: no emoji, no em-dash, no crypto jargon, no AI-tells, no "100+". Built from real promo casts and AMA threads.

Feed these to any AI drafting ZAO posts. The never-say list matters more than the phrase library.

### 65-Day Bug Fixed — AgentMail Inbox (PR #965)

The `export $(grep AGENTMAIL_API_KEY .env.local | tr -d ' ')` pattern was word-splitting and truncating the ~70-char key, causing every AgentMail call to return 403. The inbox has been dark since April 21. A memory doc (feedback_inbox_env_load_bug, 2026-05-31) captured this; the SKILL.md was never patched. Now patched: `set -a; source .env.local; set +a`. One line. Sixty-five days.
