### ZOE Evolution (5 improvements)

1. **Morning brief FOCUS** (PR #1114, Monday) -- Top-3 by deadline always surfaces first. Anything that needs Zaal never hides behind lower-priority items.
2. **Task auto-tagger** (PR #1117, merged Thursday) -- Deterministic keyword classifier assigns brand, work-type, and next owner to every task ZOE creates. 26/26 bot tests, zero human-set tags overwritten.
3. **Tier taxonomy** (PR #1194, Friday) -- ZOE's proactive candidates now carry a tier label (doc 1015 framework: life, money, relationships, health, then everything else).
4. **Work-loop silent failure fix** (PR #1189, Thursday) -- ZOE now notifies Zaal when the dispatch loop fails silently instead of continuing as if nothing happened.
5. **Quote-reply context fix** (PR #1217, Friday) -- ZOE now reads the context of Telegram quote-replies correctly and stopped inventing command conventions not in the spec.

**Bonus: ZOE escalation module** (Wednesday) -- New `bot/src/zoe/escalation.ts` with 45 tests. ZOE can now detect when a task needs to be escalated vs. handled autonomously. First concrete implementation of the wheel-and-spoke architecture (doc 989).

### Agent-Loop Rules 12-20 (All Week)

The agent-loop contract grew from 11 rules to 20. Every new rule came from a real failure or lesson during the week's work:
- Rules 12-18: Overnight loop-ops (inline gh-api edits, SHA freshness, browser headers, resource ownership, sibling loop monitoring, self-iteration)
- Rule 19: Rebase-to-new-branch pattern for collision-guard bypass (the #1117 → #1150 recovery)
- Rule 20: Never run two file-writing subagents concurrently in one clone (the #1192 PR-creation race)

These rules are now in `.claude/rules/agent-loops.md` and inherited by every future loop and ZOE automatically.
