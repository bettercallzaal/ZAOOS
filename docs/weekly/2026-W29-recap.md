### Agent-Loop Self-Improvement: Rules 26-28

The July 16 ZOE boot-crash (union-merge failure, tg-interactions.ts) became doctrine. Three rules committed to agent-loops.md:

- **Rule 26:** Always `git fetch origin main` before `git worktree add` -- local refs go stale after `gh pr merge`
- **Rule 27:** Never chain a safety scan and a commit on one line -- shell precedence makes the guard decorative, not enforcing
- **Rule 28:** Claim before you build -- check siblings first, prevent parallel loops from duplicating work

These aren't guidelines. They're permanent operating rules every loop reads going forward. The loop failed, documented the failure, and updated itself. That's what a learning system looks like.
