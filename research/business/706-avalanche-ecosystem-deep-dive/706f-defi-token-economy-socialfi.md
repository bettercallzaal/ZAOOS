## 6. Bridging ERC-20 from Base (or Ethereum) to Avalanche C-Chain

### Available Bridge Solutions

#### Option A: Avalanche Core Wallet (Native Bridge)

- **Source chains:** Ethereum, Bitcoin only (2 chains)
- **Cost:** ~$3 worth of BTC or ERC-20 per transfer
- **Security:** Updated with Intel SGX Enclave technology (2022+)
- **Speed:** Native, fast finality
- **Best for:** ETH-mainnet and BTC users

#### Option B: Avalanche Interchain Token Transfer (ICTT)

- **Mechanism:** Smart contracts deployed across multiple Avalanche L1s; leverages Teleporter for cross-chain messaging
- **Security model:** Non-upgradeable contracts; relies on source L1 validator set (unbounded number of validators)
- **Trust assumption:** ZERO additional trust introduced beyond source L1 validators (major advantage)
- **Cost:** Gas handled by relayer service (paid upfront by token deployer, not user)
- **Best for:** Avalanche L1-to-L1 transfers; future-proof interop
- **Current limitation:** No direct Base-to-Avalanche C-Chain bridge via ICTT yet (would require Base integration)

#### Option C: Third-Party Bridges (Axelar, Celer, Jumper, Wormhole)

| Bridge | Coverage | Base to Avax | Cost | Risk |
|--------|----------|-------------|------|------|
| **Axelar** | 45+ blockchains | YES (supported) | Gas fees vary | Axelar validator set (not Avalanche validators) |
| **Celer cBridge** | 40+ blockchains | YES (USDT, USDC) | Competitive | Celer security assumptions |
| **Wormhole** | 20+ blockchains | YES | Variable | Wormhole guardian set (19 guardians initially) |
| **Jumper** | 22 blockchains | YES | Multi-bridge aggregator | Depends on underlying bridge |

**Key consideration:** Third-party bridges introduce additional trust assumptions (e.g., Wormhole's 19 guardian multisig, Axelar's validator set). ICTT is more decentralized but not yet available for Base-to-Avalanche C-Chain.

### Path for $ZABAL (Base -> Avalanche C-Chain)

If $ZABAL (currently ERC-20 on Base) bridges to Avalanche C-Chain:

1. **Recommended approach:** Use Axelar or Celer cBridge (both support Base natively)
   - Axelar: 45-chain coverage, universal connections
   - Celer: Proven USDT/USDC bridging, competitive fees

2. **Cost structure:** 
   - Axelar: Varies by gas congestion; typically $20-100 per transfer depending on source/dest chain demand
   - Celer: Similar range; may be cheaper for stablecoin pairs

3. **Setup requirements:**
   - Deploy $ZABAL "remote" instance on Avalanche C-Chain (via Axelar's ITS or Celer adapter)
   - Register token on Axelar/Celer as "approved" (whitelisting step)
   - Users call native Core Wallet or third-party interface to initiate bridge

4. **Risk mitigation:**
   - ICTT is not yet available for Base-to-Avalanche (Avalanche ICTT only spans Avalanche L1s currently)
   - Use Axelar (more decentralized than Wormhole) if available; otherwise Celer is solid second choice
   - Test on testnet (Fuji) first; audit bridge contracts if bridging >$5M

**Recommendation:** Plan for Axelar or Celer as primary bridge. Monitor Avalanche ICTT roadmap for future Base support (unlikely near-term).
