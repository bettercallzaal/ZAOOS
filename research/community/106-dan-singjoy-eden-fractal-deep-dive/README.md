## Part 3: The Respect Game -- Step-by-Step Guide

### Complete Process

**Step 1: Group Formation**
- Participants are randomly assigned to Zoom breakout rooms
- Groups of 3-6 people (ensures meaningful discussion)
- Random assignment prevents cliques and ensures fresh perspectives
- If many participants, multiple rooms run simultaneously (this is the "fractal" part)

**Step 2: Prompt Introduction**
- Each session has a central prompt (e.g., "What did you do to advance governance this past two weeks?")
- All groups use the same prompt

**Step 3: Contribution Sharing (4 min each)**
- Each participant gets up to 4 minutes to describe their recent contributions
- Can include: tool development, community building, content creation, event organizing, onboarding new members, governance proposals, anything relevant to the prompt
- For ZAO context: could be music production, curation, event organizing, community support

**Step 4: Group Discussion**
- After all presentations, the group openly discusses the contributions
- Compare impact, effort, innovation, community benefit
- Collaborative, not adversarial -- the goal is fair assessment, not competition

**Step 5: Collaborative Ranking (Levels 1-6)**
- Group ranks all contributions from most impactful (Level 6) to least impactful (Level 1)
- With 4 people in a room, rankings use Levels 1-4
- With 6 people, rankings use Levels 1-6

**Step 6: Consensus Check (2/3 required)**
- At least 2/3 of the group must agree on the final rankings
- Fractalgram facilitates this via automated Telegram polls
- Discussion continues until consensus is reached
- If consensus cannot be reached, no one in the group earns Respect for that session

**Step 7: Fibonacci-Weighted Respect Distribution**
- Respect tokens distributed using Fibonacci-like sequence:

| Rank | Level | Respect Tokens |
|------|-------|---------------|
| 1st (most impactful) | 6 | 55 |
| 2nd | 5 | 34 |
| 3rd | 4 | 21 |
| 4th | 3 | 13 |
| 5th | 2 | 8 |
| 6th (least impactful) | 1 | 5 |

- Each higher rank earns ~60% more Respect than the previous
- Prevents winner-take-all while still rewarding top contributors

**Step 8: Onchain Recording**
- Results submitted onchain to Base via Fractalgram/ORDAO
- Creates immutable, transparent record of all contributions and rankings
- Respect tokens are **soulbound** (non-transferable) -- you keep them as personal reputation
- Each token has unique artwork commemorating the event

### What Respect Tokens Unlock

- **Council eligibility** -- Accumulate enough Respect to join governance councils
- **Voting weight** -- More Respect = more influence in community decisions
- **ORDAO proposals** -- Create and vote on proposals using Respect-weighted governance
- **Role progression** -- Hats Protocol integration: earn specific Respect thresholds to unlock hats (roles)
  - 5 Respect = "Rookie" hat (workspace access)
  - 50 Respect = "Rising Star" hat (leadership at events)
  - Rookie + Rising Star = "Respect Game Leader" (major leadership role)

## Part 4: Eden Fractal Governance Model -- Tripartite System

### The Three Branches

Eden Fractal implements a three-branch governance system inspired by democratic theory:

#### 1. Judicial Branch: The Respect Game
- **Function:** Peer evaluation of contributions -- the foundation of the system
- **How:** Breakout room consensus rankings (described above)
- **Output:** Soulbound Respect tokens distributed fairly via Fibonacci weighting
- **Frequency:** Bi-weekly
- **Analogy:** Jury system -- small groups of peers make nuanced assessments

#### 2. Legislative Branch: Eden+Fractal Consensus + Sages Council
- **Function:** Proposal creation, parameter voting, community direction
- **How:** Elected delegates from breakout rooms form weekly councils
- **Election process:**
  1. During Respect Game breakout rooms, each group also selects a delegate
  2. Delegates from all rooms form a council for that week
  3. Councils update each week with new delegates
  4. Recent councils cooperate to represent the community's will
- **Voting:** Primarily off-chain via Snapshot (using Respect-weighted votes)
- **Decisions:** Community agreements, parameter changes, strategic direction
- **"System 2" thinking:** Slow, deliberative, abstract

#### 3. Executive Branch: ORDAO (Optimistic Respect-based DAO)
- **Function:** Automated onchain execution of approved proposals
- **How:** OREC smart contracts on Base execute community decisions
- **Voting mechanism:** Consent-based (not majority):
  - Phase 1: YES/NO voting with 2:1 ratio required
  - Phase 2: Veto-only period (time-delayed execution allows opposition)
  - Low quorum (e.g., 5%) fights voter apathy
  - Trusts active minority; broader community can block via veto
- **"System 1" thinking:** Fast, automated, technical
- **Deployed on:** Base (Ethereum L2)

### How Decisions Flow

```
Breakout Room Discussion
        |
        v
Delegate Elected (per room)
        |
        v
Weekly Council Formed (all delegates)
        |
        v
Council Creates Proposal (Snapshot vote)
        |
        v
Community Votes (Respect-weighted)
        |
        v
ORDAO Executes Onchain (if approved)
        |
        v
Veto Period (community can block)
        |
        v
Execution Complete
```

### Synchronous Respect Trees (SRT)

A community-driven topic selection game:
- Uses earned Respect to democratically prioritize discussion topics
- Participants propose topics and vote with Respect weight
- Highest-weighted topics become the event's focus
- Ensures community attention goes where members collectively want it

### How Fractalgram Works in Practice

1. **Original version (2023, production):** A Telegram client fork that adds Respect Game features:
   - Creates automated Telegram polls for group ranking
   - Each player ranks others' contributions via poll interface
   - Requires 2/3 consensus to finalize
   - One-button submission to push scores to the web app
   - Results then posted onchain

2. **Next-gen version (2024, alpha at respect-game.vercel.app):**
   - Multiple wallet connection options
   - Custom usernames
   - Hats Protocol integration
   - ORDAO smart contract integration
   - MIT licensed (original was GPL-3.0)

### How Results Get Submitted Onchain

1. Group reaches 2/3 consensus on rankings in Fractalgram
2. Designated submitter (often a developer or leader) pushes results
3. Fractalgram sends rankings to ORDAO smart contracts on Base
4. OREC records the Respect distribution as soulbound ERC-1155 tokens
5. Each token includes unique artwork and metadata about the event
6. Results visible on-chain and via the ORDAO app at of.frapps.xyz

## Part 7: Eden Fractal & Hats Protocol Integration

### Role Progression System

Eden Fractal (via Optimism Fractal's implementation) uses Hats Protocol for automated, onchain role management tied to Respect accumulation:

| Hat | Requirement | Powers |
|-----|-------------|--------|
| **Fractalgram Certification** | Sign onchain agreement (Community Agreement Module) | Basic member access |
| **Rookie** | Earn 5 Respect tokens | Access to community workspaces |
| **Rising Star** | Earn 50 Respect tokens | Leadership role at events |
| **Respect Game Leader** | Hold both Fractalgram Cert + Rising Star | Major leadership role in the DAO |

### How It Works

- Hats bundles powers, permissions, and accountability
- Verification is automatic -- contributors level up based on onchain Respect balance
- No manual tracking needed
- Roles are revocable if conditions change (e.g., Respect decays or agreement violated)
- Uses Hats Agreement Eligibility module for onboarding tutorials

### ZAO Relevance

ZAO also uses Hats Protocol. This creates a direct integration path:
- ZAO members earning Respect in Eden Fractal events could automatically receive hats in both systems
- Cross-community reputation via Superchain ORDAO (hub-and-spoke model)
- Shared hat trees between ZAO (Optimism) and Eden Fractal (Base)
