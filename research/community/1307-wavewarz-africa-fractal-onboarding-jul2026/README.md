## What WaveWarZ Africa + ZAO Fractal Means

The proposal likely involves two overlapping concepts:

### 1. WaveWarZ Africa: African Artists in WaveWarZ Battles

African independent musicians submit songs to WaveWarZ via wwtracker.xyz — songs that would otherwise have zero distribution path into web3 music ecosystems. Hurricane (SongChainn) is the connector: he has relationships with Africa-based artists already on Solana.

Key facts:
- WaveWarZ has no geographic restriction — any artist on Solana can submit a song
- 921 songs are already in the battle pool (mostly non-African)
- Hurricane's African SongChainn artists would bring new songs + new audience segments
- Board task acc90fe2 ("Talk to Hurricane re African SongChainn artists") is DONE — Hurricane is interested

### 2. ZAO Fractal Africa Node: A ZAO Governance Chapter in Africa

The ZAO runs Eden Fractal-style governance sessions weekly (100+ sessions, 63 on-chain, 157 Respect holders). A "ZAO Africa node" would be:
- An Africa-based group that runs their own weekly Fractal sessions
- Uses the same Respect1155 governance contracts on Optimism
- Participates in WaveWarZ as a "ZAO Africa" chapter

ZAO precedent: Eden Fractal is the parent protocol; The ZAO is already a regional node of Eden Fractal. A ZAO Africa node would be a sub-node.

## Zaal's Response Framework

When Zaal reads Iman's doc, he should respond to these 5 dimensions:

### A. Artist Pipeline: Approve with Conditions

**Approve:** WaveWarZ Africa artist onboarding is low-friction. Any artist with a Solana wallet can submit.

**Conditions Zaal needs to set:**
- Hurricane must vet submissions (curator gate, same as existing artists) — otherwise WaveWarZ gets flooded with low-quality submissions
- Artists should have at least one social presence (X, Instagram, or Farcaster) for WaveWarZ to cite them
- Onboarding doc should include a clear "here's how to create a Solana wallet" for users who've never done it

**Open question for Iman:** "What's Hurricane's capacity to curate? Is he onboarding 5 artists or 50?"

---

### B. Fractal Node: Pilot First, Scale Later

**The ZAO's Eden Fractal node started with ~15 people and took 6 months to reach weekly consistency (100+ sessions).** A ZAO Africa node should follow the same minimal-viable approach:

1. Find 1 local organizer (Hurricane? Someone in his network) who commits to hosting weekly
2. Run 4 sessions with just the core group to test timing, format, tech (Telegram/Discord voice for sessions?)
3. Add Respect tracking from session 1 (on-chain = receipt from day 1)
4. Connect Africa node to the main ZAO Fractal after 4 sessions of success

**What NOT to do:** Don't build a full "ZAO Africa" brand, website, or Discord server before running 4 successful sessions. Start with a group chat and a recurring time.

**Zaal's response to Iman:** "Pilot: find 1 organizer, run 4 sessions, then scale. Who is the organizer — Hurricane, or someone he recommends?"

---

### C. Economic Participation: Same as Global

Africa node participants earn ZAO Respect via Fractal sessions exactly like Maine participants. No special economic treatment needed — the existing system already works globally because:
- Respect1155 is on Optimism (low gas)
- ZAO Fractal sessions distribute Respect via the existing scripts
- ZABAL Games (if extended to Africa node in Cohort 2) would include Africa builders

**Open question:** Does Iman's doc propose any Africa-specific incentives (bonus Respect, local prize pool)? If so, Zaal needs to decide whether to approve that.

---

### D. WaveWarZ Integration with Africa Fractal

The natural flywheel: Africa Fractal node members vote on WaveWarZ battles → surface African artists as the Africa "collective pick" each week → Farcaster cast about it.

This is a 2-step process:
1. Africa artists get songs into WaveWarZ → their community votes for them
2. Africa Fractal weekly session includes a "this week's WaveWarZ Africa picks" discussion

This is similar to how ZAO Fractals already discuss weekly WaveWarZ results.
