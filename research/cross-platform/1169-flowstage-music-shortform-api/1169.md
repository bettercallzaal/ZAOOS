# FlowStage: Music Short-Form Video API — Endpoints, Auth, and ZAO Integration

## Key Decisions / Recommendations

| Decision | Answer |
|----------|--------|
| **What is it?** | Music artist short-form video creation tool (TikTok / Reels) by MWM — NOT a scheduling platform |
| **Disambiguation** | FlowStage ≠ FlowStage.io (React component framework). FlowStage = app.theflowstage.com — two different products |
| **ZAO fit** | Narrow but real: WaveWarZ battle recap clips, COC Concertz show moments, artist marketing content |
| **Auth to use** | OAuth popup (any plan) → `fs_` key via postMessage. Manual key (Growth plan+, $$/mo) |
| **Integration effort** | Low — 7 REST endpoints, async render (30–60s), clean TypeScript types in public sample repo |
| **Blocker** | Artist audio must be uploaded to a FlowStage workspace first (no direct URL ingest); one workspace per account |

## What FlowStage Does

FlowStage (app.theflowstage.com, by MWM) turns artist audio + video footage into short-form clips styled with aesthetic templates — watermark-free exports to TikTok/Reels. Artists upload an audio track, pick an "aesthetic" (visual style / template), select a clip section, optionally layer in their own video clips, and FlowStage renders a polished short-form video in 30–60 seconds.

Core workflow:
1. Upload audio to a FlowStage workspace (done manually via the web app; no upload API as of this research)
2. Choose an `aesthetic_id` (visual template) and `audio_id` (track)
3. POST `/v1/video-edits/draft` with section timestamps, optional video clips, hook text
4. Poll `/v1/video-edits/{id}/progress` until `status === 'done'`
5. Download `url` (signed, available when `status === 'done'`; no `url` until then)

## Auth

| Method | Plan required | How |
|--------|--------------|-----|
| **OAuth popup** | Any plan (including free) | `window.open('https://app.theflowstage.com/authorize-app?app_name=<name>')` → listen for `postMessage` event from `https://app.theflowstage.com` with `type: 'FLOWSTAGE_AUTH_SUCCESS'` → payload contains `apiKey` |
| **Manual API key** | Growth plan ($$/mo) | Dashboard → `app.theflowstage.com/api-keys` → copy key |

API key format: `fs_` + 48 hex characters (minimum 51 chars total). All requests use header `X-API-Key: fs_xxx`.

**Security constants** (from sample repo config):
- `AUTH_TIMEOUT_MS`: 300000 (5 minutes — time to wait for OAuth popup response)
- Strict origin check: only `https://app.theflowstage.com` postMessages are trusted
- Do NOT log full key — sanitize to first 11 chars + `***` in any debug output

## API Reference

**Base URL:** `https://api.theflowstage.com`

### Aesthetics (visual style templates)

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/v1/aesthetics` | GET | List all aesthetic templates with full detail |
| `/v1/aesthetics/summary` | GET | Lightweight list: thumbnails + metadata counts (faster, use for picker UI) |
| `/v1/aesthetics/{id}` | GET | Full detail for one aesthetic: available videos, audios, hooks, presets |

#### `Aesthetic` shape (from `/v1/aesthetics`)
```typescript
interface Aesthetic {
  id: string
  name: string
  description: string
  thumbnail_url: string
  video_count: number
  audio_count: number
  created_at: string
}
interface AestheticSummary extends Aesthetic {}  // same, used for summary list
interface AestheticDetail extends Aesthetic {
  videos: Array<{ id: string; url: string; duration: number }>
  audios: Array<{ id: string; title: string; artist: string; duration: number }>
  hooks: string[]
  presets: string[]
}
```

### Video Edits

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/v1/video-edits/draft` | POST | Create a new video edit (optionally trigger render immediately) |
| `/v1/video-edits/{editId}` | GET | Get edit status, metadata, download URL when done |
| `/v1/video-edits/{editId}/progress` | GET | Render progress — use this for polling |

#### `CreateEditRequest` (POST body)
```typescript
interface CreateEditRequest {
  aesthetic_id: string           // required: which visual style to use
  audio_id: string               // required: which audio track
  section_start_time: number     // required: start of audio clip (seconds)
  section_end_time: number       // required: end of audio clip (seconds)
  preset_name?: string           // optional: video or slideshow preset
  hook?: string                  // optional: text overlay hook (short phrase)
  name?: string                  // optional: display name for the edit
  render?: boolean               // if true, triggers render immediately; default false
  videos?: Array<{               // optional: custom video footage to composite
    video_id: string
    start_time: number
    end_time: number
  }>
  is_flipbook?: boolean          // slideshow mode (images instead of video clips)
}
```

#### `VideoEdit` (GET response)
```typescript
interface VideoEdit {
  id: string
  name: string
  status: 'pending' | 'processing' | 'done' | 'error'
  aesthetic_id: string
  audio_id: string
  created_at: string
  updated_at: string
  url: string | null             // download URL — only populated when status === 'done'
}
```

#### `RenderProgress` (polling response)
```typescript
interface RenderProgress {
  edit_id: string
  status: 'pending' | 'processing' | 'done' | 'error'
  progress: number               // 0–100
  url: string | null             // populated when done
  message: string                // human-readable status
  error: string | null           // only on status === 'error'
}
```

## Render Polling Pattern

FlowStage renders are async: POST the draft → `status` is `pending` → poll progress → `done` in 30–60s. Sample repo uses exponential backoff:

```typescript
async function pollRenderProgress(client: FlowstageClient, editId: string): Promise<RenderProgress> {
  const MAX_POLLS = 30
  // First 10 polls: 2s. Next 10: 5s. After 20: 10s.
  for (let i = 0; i < MAX_POLLS; i++) {
    const delay = i < 10 ? 2000 : i < 20 ? 5000 : 10000
    await sleep(delay)
    const progress = await client.getRenderProgress(editId)
    if (progress.status === 'done' || progress.status === 'error') return progress
  }
  throw new Error('Render timed out after 30 polls')
}
```

When `status === 'done'`, `progress.url` holds the download link for the rendered video.

## ZAO / COC Concertz Pipeline Fit

### Strong fit
| Use case | How |
|----------|-----|
| **WaveWarZ battle recap clips** | Upload battle audio segments → pick a hype aesthetic → render 30–60s recap clip → post to TikTok/Reels via Postiz |
| **COC Concertz show moments** | DJ Zaal's set audio (clipped by zaalclip) → FlowStage aesthetic → short vertical video for Instagram Reels, TikTok |
| **Artist short-form marketing** | WaveWarZ / COC artists upload tracks → auto-generate aesthetic clip → schedule via Postiz |

### Integration with existing tools

| ZAO tool | Integration point |
|----------|-----------------|
| **zaalclip** | Livepeer/Cloudflare clip → download segment → upload video clips to FlowStage aesthetic via `videos[]` param |
| **Postiz** (zaalclip instance, `POSTIZ_API_KEY`) | FlowStage renders the video → Postiz schedules it to TikTok, Reels, X |
| **Restream** | Live show audio → FlowStage recap clip post-show |
| **Bonfire** | Episode per FlowStage clip created → knowledge graph tracks what was rendered + when |

### Limits / Gaps
- **No audio upload API**: Tracks must be manually uploaded to the workspace via web UI. Blocks fully headless automation.
- **One workspace per account**: All artists sharing one account share one audio library. Organizational challenge for multi-artist ops.
- **Plan gate on manual keys**: OAuth works on any plan, but programmatic key generation requires Growth plan.
- **No Restream/live ingest**: FlowStage is clip-based (offline render), not live streaming. Pair with Restream for live, FlowStage for post-show clips.
- **30–60s render latency**: Acceptable for scheduled posts; too slow for real-time show-night clips.

## Recommended ZAO Integration Path

1. **Auth**: OAuth popup flow (any plan) → store `fs_` key in `zaalclip` env as `FLOWSTAGE_API_KEY`
2. **Aesthetics**: Cache `GET /v1/aesthetics/summary` weekly — aesthetics don't change often
3. **Clip creation pipeline**:
   - zaalclip extracts show segment → FlowStage `POST /v1/video-edits/draft` with `render: true` → poll until `done` → Postiz schedules the `url`
4. **Rate limit guard**: Check `GET /v1/limits` before bulk renders to avoid hitting `video_edits_per_month`
