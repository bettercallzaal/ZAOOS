## 1. Purpose & Classification

SNAPS (6 repos):
- **zlank**: Platform to build and deploy Snaps without code. Lives at zlank.vercel.app. Core product.
- **zlank-snap-template**: Starter template. Hono + Vercel handler. Shows the multi-page pattern, form input + state passing + share-cast.
- **duodo-snap, nouns-snap, zabalsnap1, ltaesnap**: Snap demo instances, each a mini app you can build with zlank or customize from the template.

OTHER (1 repo):
- **farmdrop**: Maine farmers market platform. NOT a Snap. Separate product (Hannah Semler, CEO).

## 2. Stack Patterns

All 5 Snaps use **Hono + Vercel** pattern:
- `@farcaster/snap` (SDK, v1.15.1)
- `@farcaster/snap-hono` (adapter, v1.4.8)
- `@hono/node-server` (local dev)
- `@noble/curves` (signing)

Variation:
- **ltaesnap**: Omits `@hono/node-server` (uses `scripts.start` instead). Smallest footprint (29 KB).
- **nouns-snap, zabalsnap1**: Add `viem` for on-chain reads (Nouns contract, Zabal contract).

zlank itself is **Next.js 16 meta-platform** that generates Snap repos.

## 5. Cross-Dependencies

duodo-snap, nouns-snap, zabalsnap1 are **independent instances**, not forks:
- Each has own git remote (no fork/parent relationship).
- Each deployed to own Vercel domain.
- Stack identical to zlank-snap-template.

**Relationship model:** zlank-snap-template is the canonical TEMPLATE. The demo snaps are independent projects that adopt the same pattern. They are NOT generated by zlank; they are hand-built to showcase what you CAN build with the pattern.

Implication: if zlank ship logic changes, these snaps don't auto-update. They're reference implementations, not SDK dependents.

## 6. Nested Repos in ZAOOS Root

Doc 661e flagged duodo-snap/ and nouns-snap/ as nested git repos (git cruft).

Confirmed:
- `/duodo-snap/.git/config` has remote `https://github.com/bettercallzaal/duodo-snap.git`
- `/nouns-snap/.git/config` has remote `https://github.com/bettercallzaal/nouns-snap.git`

These are legitimate separate repos (not git submodules). However, they live in ZAOOS root when they could live cleanly in their own GitHub repos only (as all others do). **They should be removed from ZAOOS and accessed only via gh CLI / open links if needed as reference.**

zabalsnap1, ltaesnap, farmdrop are NOT nested locally — they exist only in GitHub.
