## 2. What to Submit (Submission Checklist)

Every finalist must submit all **4 items** to qualify for judging:

- [ ] **GitHub repo URL** — public, with a README that describes what the project does and how to run it
- [ ] **Live demo URL** — a working link (Vercel, Railway, or similar); local-only apps are allowed with a Loom if hosting is blocked
- [ ] **2-minute demo video** — uploaded to YouTube (unlisted OK), Loom, or Warpcast (see section 3 for what to cover)
- [ ] **Project description block** — paste the template from section 4 into the submission form

Submit via: **[zabalgamez.com/submit]** (Zaal/Iman to create before Aug 1)

Incomplete submissions at Aug 18 deadline = disqualified. No exceptions.

## 3. The 2-Minute Demo: Exactly What to Show

Judges watch the demo video before scoring. Keep it tight.

**Format (total: 120 seconds)**

| Segment | Time | What to say |
|---------|------|-------------|
| **Hook** | 0–15s | One sentence: what problem you solved and for whom |
| **Demo** | 15–90s | Screen share of the live app. Click through the main user flow. No slides. Show real data if possible. |
| **ZAO/Web3 angle** | 90–105s | How does this connect to WaveWarZ, Fractal Governance, Farcaster, or onchain mechanics? Even tangentially. |
| **What you'd build next** | 105–120s | One sentence: the most important thing you'd add with another month. |

**Common mistakes to avoid:**
- Showing code instead of the running app
- Spending 45+ seconds on backstory before showing anything
- "I'm still working on it" — show what exists, not what you plan

## 5. Scoring Rubric (100 points total)

Judges score each project on these 5 dimensions. Fractal Respect vote applies as a 50% weight after expert scores are in.

| Dimension | Points | What judges look for |
|-----------|--------|----------------------|
| **Shipped** | 25 | Does it run? Can a user actually use it right now? No points for wireframes or repos with no deployable code. |
| **Problem clarity** | 20 | Is it obvious what problem this solves and who benefits? Builder should not have to explain it. |
| **ZAO ecosystem alignment** | 20 | Does this make WaveWarZ, Fractal, ZABAL, or ZAO better? Even partial alignment scores here. |
| **Technical ambition** | 20 | Did they push beyond copy-paste tutorials? Onchain mechanics, real API integrations, and novel UX score high. |
| **Demo quality** | 15 | Does the 2-min video cover the required segments without going over? Clear, confident walkthrough. |

**Fractal Respect weight:** After expert panel scores (100 pts) are in, the final score = 50% expert + 50% Fractal Respect vote. Fractal vote is live during the X Space showcase.

## 9. Builder FAQ

**Q: My project isn't finished. Can I still submit?**
A: Submit what works. Judges score on "does it run" — a partial app with a working core feature beats a fully-designed app that doesn't run.

**Q: Do I have to use a ZAO or WaveWarZ API?**
A: No. Ecosystem alignment is one of 5 dimensions, worth 20 points. A great project with no ZAO integration still competes.

**Q: Can I submit a project I started before ZABAL Games?**
A: Yes, as long as you made meaningful progress during the season (attended at least 3 workshops). Judges can ask.

**Q: What if I miss the Aug 18 deadline?**
A: No extensions. The showcase schedule and Fractal vote don't allow it.

**Q: Can two builders submit the same project as a team?**
A: Yes. The prize goes to both builders (split ZABAL, shared SOL, one COC slot split by negotiation). Both must be registered ZABAL Games participants.
