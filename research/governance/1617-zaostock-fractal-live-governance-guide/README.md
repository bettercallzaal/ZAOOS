# 1617 — ZAOstock Oct 3 Fractal Governance: Live Event Voter Guide

> **Who this is for:** Any ZOR holder attending ZAOstock in-person OR participating remotely. The Oct 3 governance session is the first live IRL Fractal Democracy session to include formal proposal voting alongside the weekly Respect Game.

---

## What Makes ZAOstock Governance Different

Every Thursday, ZAO runs a standard Fractal Democracy session via Telegram voice or Juke (doc 1475, doc 1553). ZAOstock Oct 3 is different:

| Regular Thursday | ZAOstock Oct 3 |
|---|---|
| Telegram voice / Juke | In-person + Telegram hybrid |
| ~10–20 regular participants | Open to all ticket holders (~50–150 potential new voices) |
| Informal proposal discussion | Formal vote on submitted proposals (doc 1568 template) |
| Remote-only | IRL-primary, with remote stream option |
| ~60 minutes | ~90 minutes (extended for proposals) |

The ZAOstock session produces the **highest governance weight of any session in Season 9** because IRL presence = maximum weight in the Fractal Democracy scoring. (Doc 1253: ORDAO mechanics award more weight to synchronous IRL participation.)

---

## The Governance Agenda (Oct 3, ZAOstock)

### Block 1 — Fractal Democracy Game (~45 min)

Standard ORDAO/Optimystics format. Every participant gets 3 minutes to speak about their contribution since the last session. Groups of 4–6 rank each other. Top-ranked advances to the Respect table.

**IRL participants:** Join the designated governance space at the festival. ZAO team will have this marked.

**Remote participants:** Dial in via the session Telegram voice link (ZOE posts 30 minutes before start).

**What counts as "contribution" at ZAOstock:**
- Helping run ZAOstock: volunteer work, artist liaison, sponsor wrangling
- ZAO product contributions: code shipped, research docs, WaveWarZ battles run
- Community growth: onboarding new members, hosting WaveWarZ battles, running a Fractal node
- Creative work: music released that brought people to the ZAO ecosystem

### Block 2 — Proposal Votes (~30–45 min)

Submitted proposals (deadline: Sunday Sep 28) are voted on in sequence. See below for how to submit and vote.

### Block 3 — Session Archive (~15 min)

ZOE posts the session archive immediately after (doc 1540 template). TX hash submitted to OREC contract. ZOR distribution to top-ranked participants runs post-session per ORDAO mechanics.

## How to Submit a Proposal

Use the template from doc 1568. Key fields:

```
GOVERNANCE PROPOSAL — ZAO Fractal
Target vote: [x] ZAOstock Oct 3 live session
Submitted by: [your name / wallet]
Date: [before Sep 28]

TITLE: [One clear sentence]
THE QUESTION: [A clear yes/no OR two options]
CONTEXT: [Why this matters, what happens if we don't decide]
```

**Where to submit:**
1. DM Zaal on Telegram with your proposal
2. OR post in ZAO main Telegram with subject: `[GOV PROPOSAL]`
3. OR email zaal@thezao.com with subject: "ZAOstock Oct 3 Governance Proposal"

**What makes a good proposal:**
- One clear question the group can answer with A, B, or No action
- Context anyone can understand in 90 seconds
- A reason why Oct 3 is the right time to decide (vs. any Thursday)

**Example good proposals:**
- "Should ZAO commit to running a second annual fractal expansion node (outside Maine) by Dec 2026?"
- "Should ZOR holders vote to allocate 10% of the ZAOstock payout to the WaveWarZ Africa charity fund?"
- "Should ZAO publish a public Season Report after Season 9 ends?"

**Example proposals that don't fit:**
- Operational decisions Zaal can make directly ("should we switch to a new calendar app?")
- Proposals without a clear action option ("what should we do about distribution?")

## How to Vote — IRL (Attending ZAOstock)

**Step 1: Know your ZOR balance before Oct 3**

Check your balance in advance. You'll need to show proof of ZOR holding to vote on formal proposals.

- Blockscout: optimism.blockscout.com → search `0x9885CCeEf7E8371Bf8d6f2413723D25917E7445c` → Holders
- ZAO dashboard: zao.frapps.xyz/fractals (if Hurricane's ZOR holder view is live)
- Mobile: Cast your address into the "My ZOR" field in MetaMask on Optimism Mainnet

**Step 2: Join the governance space at ZAOstock**

The schedule will mark when and where. Look for the "ZAO Governance" signage.

**Step 3: Participate in Fractal Democracy**

Speak your 3-minute contribution piece. Be specific — what did you do since the last session you attended?

**Step 4: Vote on proposals**

For each proposal:
- Facilitator reads the question aloud
- Options are displayed
- IRL vote: raise hand (or designated voting method — TBD by facilitator)
- Your ZOR balance determines your governance weight (ORDAO mechanics)

**Tie-breaking:** If a proposal vote is split 50/50, it defaults to "no action" (status quo). A second vote may be called at a future Thursday session.

## How to Vote — Remote (Not Attending ZAOstock)

ZOR holders not attending IRL can still participate.

**Option A: Join via Telegram voice (synchronous)**
- ZOE posts the session link ~30 min before start
- Join Telegram voice on ZAO main group
- You can speak during Fractal Democracy block + vote on proposals by responding in chat ("Option A" or "Option B")
- Remote votes carry less weight than IRL in ORDAO mechanics, but they count

**Option B: Submit a vote intention in advance (asynchronous)**
- Before Sep 30, send Zaal or ZOE your vote preferences for each proposal
- Format: "For proposal [title]: I vote Option A. Reason: [one sentence]"
- Zaal will read your proxy vote aloud during the session
- Weight is counted at reduced ORDAO remote weight

**Option C: Snapshot (gasless, if set up)**

If a proposal requires a formal on-chain signal, a Snapshot vote may be posted:
- URL: snapshot.org/#/zaofractal.eth (verify this is the correct space with Zaal before voting)
- Vote window: typically 48 hours
- Gasless on Optimism Mainnet — no ETH required to vote
- ZOR balance at snapshot block is your voting power

Snapshot is used for proposals with significant consequences (treasury decisions, major partnerships). Day-to-day governance runs via Fractal Democracy session, not Snapshot.

## For Facilitators: Running the ZAOstock Session

If you're facilitating (or helping Zaal facilitate), the key differences from a regular Thursday:

1. **Larger group → more rooms.** At 30+ participants, you'll need 5–8 rooms of 4–6. Assign groups in advance if possible.
2. **New attendees need a 2-min explainer.** Add a brief "what is Fractal Democracy" before the game starts. Use the one-paragraph version from doc 1583.
3. **Proposal reading takes time.** Budget 5 minutes per proposal — 1 min to read, 1 min Q&A, 1 min vote, 1 min announce + record.
4. **Record the session.** ZAOstock will have media capture; ensure the governance space is included in the recording plan. This session will be cited as a milestone.
5. **ZOE support.** ZOE will be monitoring Telegram during the session. Tag @zaoclaw_bot if you need a stat lookup (current ZOR balances, session archive template, etc.).

Full facilitator guide: doc 1565 (Fractal Democracy Facilitator Guide).
