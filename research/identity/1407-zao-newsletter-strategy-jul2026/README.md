## Why a Newsletter Strategy Exists

The ZAO newsletter is referenced in at least 10 prior ZAOOS docs as a distribution channel for press releases, ticket launches, Mirror articles, post-event recaps, and partner announcements. But no doc defines what the newsletter IS, who it's for, how often it publishes, or how ZOE helps run it.

This doc fixes that.

**The core reason to prioritize the newsletter:** It's ZAO's only owned audience that survives platform algorithm changes. X followers can be shadow-banned. Farcaster reach fluctuates. YouTube subscribers don't guarantee views. An email list is direct: 100% deliverability to subscribers who opted in.

## Part 4: ZOE Newsletter Automation

**ZOE's role in newsletter:**
1. **Draft:** ZOE writes full draft using template + current API stats + recent ZAOOS data
2. **Alert:** ZOE sends draft to Zaal via Telegram with "Newsletter draft ready — approve to send by [date]"
3. **Stats pull:** ZOE pulls live numbers from wavewarz.info/api/public/stats before each draft
4. **Schedule:** ZOE knows the Issue Calendar above and initiates drafts 48h before send date
5. **Does NOT send:** ZOE never sends the newsletter autonomously — Zaal approves in Mailchimp/Substack before sending

**ZOE draft template (for any issue):**

```
[TRIGGER: N days before Issue N send date]

DRAFT: ZAO Brief Issue N — [Month Year]

Subject: [Subject line]

Hi [First Name],

[Opening hook — 1-2 sentences. Make it specific, not generic.]

[Main content block 1 — 100-150 words]

[CTA 1 — link + 1 sentence]

[Main content block 2 — 100-150 words]

[CTA 2 — link + 1 sentence]

[ZAOstock/Season 3/WaveWarZ sidebar — 50 words max]

[Footer — always include:
- Unsubscribe link
- "Forward to a musician or crypto friend"
- wavewarz.info | zao.community | @bettercallzaal
]
```
