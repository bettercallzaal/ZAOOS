## 1. Spotify — Countdown Pages (May 2025)

**What it does:** When an artist schedules a release in Spotify for Artists, Spotify auto-generates a public "Countdown Page" — a dedicated pre-release landing spot that fans can save. Saved fans receive a push notification at drop time. The page also feeds a real-time "most-anticipated releases" chart, giving high-save albums organic editorial lift before they even exist. No extra artist setup required; the page is generated from scheduled release metadata.

**Pillar served:** Artist Development + Culture. Pre-release hype loops stay on-platform rather than bleeding to Instagram/X.

**ZAO adaptation:** ZAO artists submit tracks via `SongSubmit`. Add a `release_date` field (optional). When set, generate a community-facing countdown card — rendered in the main feed (`src/components/social/`) and in the radio queue header (`src/providers/audio/PlayerProvider.tsx`). Members who "save" a drop get a Farcaster notification at release via `src/lib/farcaster/neynar.ts`. Top-saved upcoming releases get a featured slot in the ZAO Radio curation weight (`src/lib/music/curationWeight.ts`) — pre-release saves become a Respect-adjacent signal.

**Related research:** Check `research/050-the-zao-complete-guide/` (artist onboarding flow) and `research/051-zao-whitepaper-2026/` (artist development pillar).

## 2. SoundCloud — Fan Support Tipping (October 2025)

**What it does:** Part of SoundCloud's All-in-One Artist Subscription. A "Support" button appears on artist profiles. Fans choose a custom amount ($1–$1,000) or preset tier ($5/$10/$50), pay via card/PayPal/Apple Pay/Google Pay. SoundCloud takes **zero commission** — 100% reaches the artist via Adyen. A supporter leaderboard surfaces top tippers publicly on the artist's profile. Artists are notified per tip; payouts are monthly above a $10 floor.

**Pillar served:** Revenue (primary) + Culture. The public leaderboard transforms a private financial act into a visible status signal.

**ZAO adaptation:** Wire tipping into the existing Respect system. A tip triggers both a payment (Stripe or on-chain) and a Respect event — making financial support governance-legible. The leaderboard maps naturally onto the existing admin/profile infrastructure (`src/components/`). The key ZAO twist: tipping during a live Space or while a track plays in ZAO Radio carries a 1.5× Respect multiplier — contextual tipping is more meaningful. Target files: new `src/app/api/revenue/tip/route.ts`, `src/components/music/TipButton.tsx`, `src/lib/db/supabase.ts` (new `tips` table).

**Related research:** `research/` docs on revenue splits (Apr 01–04 entries cover waterfall, Shares, Fan Tokens).

## 3. GitHub — Enterprise Teams / Cascading Roles (October 2025)

**What it does:** GitHub Enterprise now lets owners define role assignments once at the enterprise layer; those assignments cascade automatically to all organizations and repos beneath. A new "Enterprise Security Manager" role (and others) can be granted to cross-org teams in a single UI action — no per-org duplication. Ruleset bypass lists can reference enterprise teams, so permission changes propagate in real time. Single source of truth; zero drift between orgs.

**Pillar served:** Governance. The core insight is **inheritance over repetition** — define roles at the highest relevant layer and let them flow down.

**ZAO adaptation:** ZAO's three-tier governance (ZOUNZ on-chain → Snapshot → Community Proposals) has the same fragmentation problem. Admin FIDs are currently hardcoded in `community.config.ts`. Introduce a `roles` table in Supabase: `{ fid, role: 'admin' | 'curator' | 'moderator' | 'member', granted_by, granted_at }`. Roles resolved server-side in `src/lib/auth/session.ts` at login — session carries `role` claim. Middleware (`src/middleware.ts`) can gate routes by role without per-route checks. `community.config.ts` becomes the seed definition only; runtime state lives in the DB. This makes role grants auditable, revocable, and eventually governable via Snapshot vote.

**Related research:** Governance architecture in `research/050-the-zao-complete-guide/` and ZOUNZ contracts in `src/lib/zounz/contracts.ts`.
