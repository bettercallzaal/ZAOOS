## 1. Tidal — Upload + Spotlight (November 2025)

**What it does:** Tidal Upload lets independent artists push tracks directly into Tidal's catalog in ~60 seconds — no distributor, no upfront cost, up to 200 tracks, with privacy controls and 30-day stream stats. The catch is intentional: uploaded tracks earn **no royalties**. Instead, Tidal pairs Upload with "Spotlight" — an editorial program where curators feature the best uploads on playlists and the homepage. Selected artists receive **daily Cash App payouts** tied to editorial placement, not stream counts. A companion $100K contest (Upload Headliners) ran through December 2025 with 10 winners.

**Pillar served:** Artist Development + Revenue. The design decision is radical: editorial attention is the monetization unit, not streams. Attention as currency removes the "I need a million plays" ceiling for emerging artists.

**ZAO adaptation:** ZAO already has a curation weight system (`src/lib/music/curationWeight.ts`) and a SongSubmit flow. Add a "ZAO Spotlight" queue: a weekly curated playlist where the ZAO council (admin FIDs in `community.config.ts`) nominates tracks. Featured artists receive a Respect bonus + a Tip Pool allocation funded by community governance. This collapses "editorial" + "revenue" + "governance" into one weekly ritual. Target files: `src/lib/music/curationWeight.ts` (add spotlight multiplier), `src/app/api/music/spotlight/route.ts` (new nomination endpoint), `src/components/music/SpotlightCard.tsx`.

**Related research:** `research/050-the-zao-complete-guide/` (artist onboarding), `research/051-zao-whitepaper-2026/` (Revenue pillar). Cross-references Apr 06 entry (SoundCloud tipping — the Tip Pool feeds Spotlight payouts).

## 2. Lens Protocol — V3 Groups with On-Chain Vouching (April 2025)

**What it does:** Lens V3 launched on Lens Chain mainnet (zkSync-based L2, gasless for users) with Groups as a first-class social primitive. Group owners set programmable Rules for membership: admin approval, a joining fee, token/NFT gate, or — most interesting — a **vouching system** where existing members explicitly endorse newcomers. All rules are composable and on-chain. 650,000 profiles and 125GB of social graph data migrated to Lens Chain at launch.

**Pillar served:** Governance + Culture. Vouching transforms passive membership into active social accountability. It's the on-chain equivalent of a record label A&R — insiders stake their reputation on new entrants.

**ZAO adaptation:** ZAO is already a gated Farcaster client (token-gated, Respect-based). Add a **Vouching layer to onboarding**: when a prospective member doesn't yet hold the required Respect/tokens, an existing member can vouch for them — staking a small Respect bond. If the vouched member earns positive Respect within 30 days, the bonder gets a multiplier. If they're ejected or go inactive, the bond is slashed. This creates a social recruitment incentive without requiring treasury funds. Target files: `src/lib/gates/` (add vouching check to gate logic), `src/app/api/auth/vouch/route.ts` (new), `src/components/admin/VouchQueue.tsx` (admin approval UI).

**Related research:** `research/050-the-zao-complete-guide/` (gate logic), `community.config.ts` (admin FIDs control the vouch approval queue). Governance pillar overlaps with ZOUNZ on-chain voting already implemented.

## 3. Notion — Agents (Notion 3.0, September 2025)

**What it does:** Notion 3.0 rebuilt its AI layer as autonomous Agents. Anything a human can do in Notion, an Agent can do on request: scan hundreds of database pages, bulk-update records, pull in external data (GitHub issues via search), trigger Zapier/Make webhooks, and run multi-step tasks for 20+ minutes with persistent memory. The interaction model is outcome-based: describe what you want ("find all At Risk projects and draft a summary"), not the steps. Notion 3.2 (January 2026) extended Agents to mobile and added auto-model selection across GPT-5.2, Claude Opus 4.5, and Gemini 3.

**Pillar served:** Governance + Artist Development. The "delegate an outcome, not an action" UX is the key design decision. Compare to Zapier-style trigger/action automation — Agents handle ambiguity and multi-step reasoning that rigid automations can't.

**ZAO adaptation:** ZAO already has a STOCK agent and FESTIVALS agent design spec (research docs 293–294) and an agent squad dashboard (merged PRs #131–#132). The Notion pattern points to the right interaction model for these agents: **natural language governance tasks**. Example: "STOCK, find all proposals with 800+ Respect that haven't been published yet and draft cross-platform posts for each." Target files: `src/components/admin/AgentDashboard.tsx` (add natural language task input), `src/app/api/agents/task/route.ts` (new — accepts NL instruction, routes to correct agent), `src/lib/agents/` (new domain). The agent should return a structured plan for human approval before executing.

**Related research:** `research/293-multi-agent-tools/` and `research/294-event-coordinator-ai-agents/` are directly on point. `research/154-skills-commands-master-reference/` covers the existing `/vps` and agent squad commands.
