## Part 2: Event Design

### Battle Format

| Element | Details |
|---------|---------|
| Battle type | MAIN event |
| Number of battles | 3-5 (confirm with Hurricane — standard MAIN is 3-4 battles) |
| Artists | 2-4 WaveWarZ artists (already rostered) + 2-4 Juke artists (need to onboard via doc 1302) |
| Duration | 24-48h per battle (MAIN event format) |
| SOL pool | Market-determined |
| Loser earns | Standard 1.73% of buy-side pool per battle appearance |
| Charity | Optional — if Juke wants a community battle component, apply doc 1410 protocol |

### Artist Selection

**WaveWarZ side:** ZAO selects 2-4 current WaveWarZ roster artists. Prioritize artists who:
1. Have active Audius catalogs
2. Have competed in MAIN events before
3. Have agreed to participate (ZAO DMs via artist roster — doc 1342)

**Juke side:** Juke nominates 2-4 artists from their platform. Before the event:
1. Nominated Juke artists must complete WaveWarZ onboarding (doc 1302): Audius setup, Phantom wallet, registration
2. ZOE sends doc 1302 link to Juke artists via Juke's intro channel
3. If artists aren't onboarded by Aug 10 (5 days before): event proceeds with only WaveWarZ-rostered artists

### Battle Format Variation: "Battle the Platform" Option

Alternative angle (discuss with Juke in setup call):  
"WaveWarZ artists battle Juke artists — winner's platform gets bragging rights for the month."  
This creates a narrative hook ("music discovery platforms go head-to-head") that neither WaveWarZ nor Juke alone can generate.
