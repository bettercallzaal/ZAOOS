## What Worked

1. **Workshop-first format:** 28 workshops in one month is a strong proof point that builders show up for learning + peer review, not just hackathon prize chasing.

2. **Empire Builder cross-platform:** Adrian's tokenless empire model unlocked a gamification layer (leaderboard, Empire Builder API) without requiring a token launch — exactly the builder-friendly environment ZABAL needs.

3. **Bonfires knowledge graph:** The ZABAL Bonfire agent (trained on ZAO ecosystem context) showed that a knowledge graph could serve as institutional memory for a distributed builder cohort — a pattern ZAO can package for other DAOs.

4. **Guest-facilitator model:** External guests (Mental Wealth Academy, FounderCheck, Apna Coding, Bonfires) drew new builders into the ZAO ecosystem who wouldn't have found it otherwise.

5. **Musician track differentiation:** Running a musician track alongside the builder track created cross-pollination: musicians built mini-apps, builders understood music economics.
